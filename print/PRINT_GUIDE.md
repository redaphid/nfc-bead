# Print Guide — Multi-color NFC Bead on Centauri Carbon 2

End-to-end workflow for printing an NFC bead in two filament colors (red bodies + black decoration) on a Centauri Carbon 2 with Elegoo Slicer.

This guide assumes the **Centauri Carbon 2 is multi-filament** — the printer auto-feeds different filaments mid-print. **Do NOT use M600 / manual filament-swap workflows on this hardware.** The slicer assigns filaments per part and the printer handles every transition automatically.

## Generating the STLs

The bead pipeline produces 3 separate STL files per charm:

- `Bottom.stl` — bottom half (NFC pocket recess + pegs). **Print-flipped on export** so the silhouette face is on the build plate and pegs point up.
- `Top.stl` — top half body (peg holes; outer face is the show face).
- `Decoration.stl` — raised relief on the top half (spiral / emboss / etc.). Thin ribbon, ~0.5 mm tall.

Generated by `.claude/skills/bead-stl-export/export.py`:

```python
exec(open(r"<repo>/.claude/skills/bead-stl-export/export.py").read())
```

Outputs land in `tmp/stl_export_<timestamp>/`. The script strips any architect aesthetic / debug overlays first, validates dimensions and manifoldness, and applies a deterministic per-part print-orientation flip (`EXPORT_FLIP_X_DEG` in the script tunables).

If `Top` and `Decoration` are still separate objects in the live Blender scene, the export produces them as separate STLs — that's what you want for multi-color printing. If they've been boolean-UNIONed, you only get a unified `Top.stl` — use that for single-color printing only.

## Elegoo Slicer setup (one-time)

1. Top-right printer dropdown → select **Centauri Carbon 2**.
2. Open the **Filament Settings** panel (the colored swatches on the left).
3. Confirm at least 2 filament slots are populated. The slot number maps to a physical position in the multi-filament feeder.
   - **Slot 1**: PLA Red (or whatever red filament is loaded in feeder position 1)
   - **Slot 2**: PLA Black (or whatever black filament is loaded in feeder position 2)

When the slicer says "Filament 2," the printer pulls from feeder slot 2 — no operator intervention.

## Per-print workflow

### 1. Import all three STLs

Drag `Bottom.stl`, `Top.stl`, `Decoration.stl` onto the build plate in one drop. They'll appear in the object list as three separate objects, each at their canonical world position (matching how they came out of Blender).

### 2. Group `Top` + `Decoration` as one multi-part object

- Select both `Top` and `Decoration` in the object list (Ctrl-click).
- Right-click → **Combine into one object** (or "Merge as parts" — Elegoo wording varies by version).
- They become one object with two named parts. The decoration is now locked in place on the top body and will move with it during plate-arrange operations.

`Bottom` stays as its own separate object on the plate.

### 3. Assign filaments per part

Click each part in the object list and set the filament number:

| Part | Filament | Color |
|---|---|---|
| `Bottom` | 1 | red |
| `Top` (part of merged object) | 1 | red |
| `Decoration` (part of merged object) | 2 | black |

### 4. Slice settings

In the **Process** panel:

- **Layer height: 0.12 mm.** The decoration is only ~0.5 mm tall, so it spans about 4 layers. Thicker layers will read as a stair-step instead of a smooth raised feature.
- **Infill: 100%.** These parts are tiny; sparse infill saves no print time and weakens them.
- **Supports: off.** Print orientation is chosen so nothing overhangs — bottom's pegs grow upward from the puck (no overhang), top's outer face has nothing past the spiral.
- **Wipe / prime tower: ON.** The slicer default size (~35×35 mm) is fine. The tower handles nozzle-purging during filament changes — no operator action needed.

### 5. Slice and verify

Click **Slice**, then in the preview scrub the **layer slider** to where the decoration starts (~2.5 mm Z). At that height you should see the layer split into:

- Red regions (the top body's outline at that Z)
- Black regions (the decoration's outline at that Z)

If you see only ONE color at that height, the per-part filament assignment didn't take — re-check step 3.

### 6. Print

Send the gcode to the printer. The Centauri Carbon 2 will:

- Print `Bottom` entirely in red (Filament 1) — no swaps needed.
- Print `Top` body in red until the layer where the spiral starts.
- Auto-feed Filament 2 (black), purge into the wipe tower, and lay down the black decoration.
- Continue alternating between red and black filament if the spiral region overlaps any later red layers (for rezz it doesn't — the spiral is the topmost feature).

Total print time for a rezz bead at 0.12 mm / 100% infill: **~30–45 minutes per half** including the wipe tower.

## After printing

1. Drop the **NTAG215 sticker** (10 mm dia) into the bottom half's NFC pocket recess. It should sit flush.
2. Press the top half down onto the bottom half — the 3 pegs friction-fit into the matching peg holes. No glue.
3. String through the 2 mm hole in the head of the bead.

If the press-fit feels loose: increase `PEG_CLEARANCE` in the build script (currently 0.1 mm per side); rebuild and reprint. If too tight: decrease.
